# v3.4.2 — Clasificador de Documentos Histórico

## Objetivo

Endurecer el histórico dry-run (`generar_historico_dryrun.py`) añadiendo un clasificador
local que separa documentos fiscalmente relevantes de "basura fiscal" (diseños, presupuestos,
CVs, tarifas, mandatos SEPA, informes SEO, pedidos, albaranes, justificantes bancarios).

| `presupuesto` | "presupuesto", "quotation", "proforma", "oferta económica" |
| `tarifa` | "tarifa", "price list", "tarifario", "rate card" |
| `diseno_arte` | "arte final", "mockup", "proof", "arte para imprimir" |
| `mandato_sepa` | "mandato sepa", "sepa core", "autorizacion de adeudo" |
| `justificante_bancario` | "justificante transferencia", "aviso giro bancario", "remesa" |
| `informe_seo` | "semrush", "ahrefs", "informe seo", "keyword ranking" |
| `cv_documento_personal` | "curriculum vitae", "vida laboral", "datos personales" |
| `pedido_albaran` | "albaran", "delivery note", "purchase order", "pedido no" |
| `desconocido` | Ningún keyword coincide |

**Prioridad de detección:** los tipos NO fiscales específicos se evalúan primero para
evitar falsos positivos. Si `factura` tiene score ≥ 3 (muchos keywords coinciden), 
puede sobreescribir un tipo no fiscal con baja puntuación.

## Score de confianza fiscal

### ALTA
- tipo = `factura` o `recibo`
- Y señales fiscales ≥ 3 (CIF/NIF, IVA, base imponible, etc.) + total o num_factura extraído + sin señales negativas
- O señales ≥ 2 + proveedor conocido (no desconocido) + datos numéricos + señales negativas ≤ 1

### MEDIA
- tipo = `factura` o `recibo`
- Con señales fiscales ≥ 1, o total extraído (proveedor conocido), o num_factura detectado
- O tipo `desconocido` con señales ≥ 2 y datos numéricos → reclasificado como `factura`

### BAJA
- Tipo en: `presupuesto`, `tarifa`, `diseno_arte`, `mandato_sepa`, `justificante_bancario`, `informe_seo`, `cv_documento_personal`, `pedido_albaran`
- O tipo `desconocido` sin señales fiscales
- O tipo `factura`/`recibo` sin señales ni datos numéricos

**Override:** si confianza es ALTA pero señales negativas ≥ 2, baja a MEDIA.

### Señales fiscales positivas (aumentan confianza)
`cif`, `nif`, `vat`, `iva`, `igic`, `irpf`, `base imponible`, `tipo impositivo`,
`cuota tributaria`, `total factura`, `importe total`, `total a pagar`,
`numero de factura`, `factura num`, `nº factura`, `fecha de factura`,
`fecha emision`, `razon social`, `domicilio fiscal`, `euros`, `eur`

### Señales negativas (reducen confianza)
`arte final`, `mockup`, `diseno`, `curriculum`, `sepa mandate`, `mandato sepa`,
`seo report`, `semrush`, `ahrefs`, `presupuesto`, `quotation`, `proforma`,
`albaran`, `delivery note`, `aviso giro`, `giro bancario`, `extracto bancario`

## Lección aprendida (L-016)

La query `has:attachment filename:pdf` captura muchos documentos no fiscales
(diseños, presupuestos, CVs, mandatos SEPA, informes SEO). La solución no es
estrechar la query Gmail (riesgo de perder facturas reales) sino clasificar
después con un score de confianza basado en keywords del nombre del PDF,
remitente, asunto y texto extraído por pdfplumber.
